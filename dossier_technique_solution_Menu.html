Solution de Menu Digital & Commande QR Code
Date : 21 Janvier 2026
Émetteur : Ny Avo RAFIDISON
Destinataire : ORTUS Sarlu (Madagascar)

Ce document constitue la réponse technique officielle aux éléments requis pour la commercialisation de la solution "Menu Digital" à Madagascar. Il définit le périmètre fonctionnel, les prérequis techniques et le cadre de maintenance.

1. Fiche Produit & Périmètre

2.1 Identité du Produit

  • Nom suggéré : Gourmet.mg (ou Digital Menu MG)
  • Description : Interface web de commande instantanée pour restaurant, accessible via QR Code, connectée en temps réel à un écran cuisine.
  • Proposition de valeur (Pitch) : Une solution qui supprime les coûts de réimpression des menus, accélère le service et sécurise l'encaissement, sans nécessiter d'investissement matériel coûteux.

2.2 Périmètre Fonctionnel (Inclus)

La solution actuelle, fonctionnelle et testée, comprend :

  • Menu Client : Affichage catégorisé (Entrées, Plats, Boissons) avec photos, descriptions et prix en Ariary.
  • Prise de commande : Panier virtuel avec calcul automatique du total (MGA).
  • Interface Cuisine (KDS) : Réception des commandes en temps réel avec alerte visuelle. Gestion de file d'attente.
  • Gestion des statuts : Fonctionnalité "Servi/Terminé" avec archivage local (mémoire cache) pour éviter les doublons.
  • Back-Office : Gestion autonome des prix et produits via Google Sheets (Excel).
  • Export Comptable : Journal des ventes automatique et historisé.

Exclusions (Non inclus à ce jour) : Paiement mobile en ligne (MVola/Orange Money), Gestion de stock décrémentale, Impression ticket thermique.

2. Prérequis Techniques (Madagascar)

Élément Spécification Requise
Connexion Internet Requise (3G/4G stable ou ADSL/Fibre). Solution à faible consommation de bande passante (fichiers textes légers).
Matériel Restaurant 1 Smartphone ou Tablette (Android ou iOS) pour la cuisine. N'importe quel modèle avec un navigateur web (Chrome/Safari) suffit.
Infrastructure Solution "Cloud" reposant sur l'écosystème Google (Drive, Sheets). Sécurité assurée par les serveurs Google.
Mode Dégradé En cas de coupure Internet : Le menu reste consultable si déjà chargé, mais l'envoi de commande nécessite le rétablissement du réseau.

3. Processus Standard d'Installation

Temps estimé par établissement : 45 à 60 minutes.

  1. Configuration Compte : Création d'un compte Google dédié au restaurant (ex: resto.nom@gmail.com).
  2. Base de données : Copie du fichier maître "Google Sheet" sur le Drive du client.
  3. Saisie Menu : Remplissage des plats, prix (Ariary) et liens photos.
  4. Déploiement Script : Installation du code serveur (Apps Script) via copier-coller (procédure fournie).
  5. Génération QR : Création du QR Code unique lié à l'URL générée.

4. Recommandations Économiques

  • Modèle : Vente "Pack Installation" (Paiement unique) + Contrat de Maintenance optionnel.
  • Devise : Ariary (MGA).
  • Liberté tarifaire : ORTUS Sarlu dispose de la liberté totale de fixation des prix finaux selon la typologie du client.
